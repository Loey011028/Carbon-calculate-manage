# 🌿 绿厂工业供应链碳 footprints 智能核算管理平台 (Carbon Precision)

本平台是一款针对工业制造、绿色工厂（Green Factory）及供应链端设计的**高保真纯静态静态碳排放核算管理系统原型**。系统完全基于静态 HTML、JavaScript (Mock 共享数据库) 和 Tailwind CSS 构建，完美还原了企业级碳管理系统在 **ISO 14064-1:2018** 和 **GHG Protocol** 标准下的核心核算闭环及版本审计控制流。

---

## 📂 项目文件架构与业务分工

项目由 7 个核心业务视图模块、1 个共享 Mock 数据内核组成。汇总模块作为主壳体通过高保真 Iframe 控制流对其他 6 个核算子页面进行统一的管理。

```bash
├── index.html                  # 核心汇总主壳体（整合 6 大核算模块，处理全局状态与面包屑导航）
├── boundary.html               # 1. 核算边界管理（范围一/二/三划定、标准边界覆盖率 BCR 核算统计）
├── source.html                 # 2. 排放源管理（物理设备/流向配置、AD 活动数据与二级分类绑定）
├── datasource.html             # 3. 数据源管理（表计、手工填报、API 自动归集、高频点位校验控制）
├── model.html                  # 4. 核算模型管理（公式、计算精度控制、多版本回滚与合规评分引擎）
├── factor.html                 # 5. 因子库管理（温室气体排放因子归集、单因子/因子组级联控制流）
├── gwp.html                    # 6. GWP管理（全球变变暖潜势温室气体参数库，AR4/AR5/AR6 跨版本对比控制）
└── shared-mock-data.js         # 🛠 核心共享Mock数据库机制（提供跨页面、跨 Iframe 的数据闭环与高保真计算逻辑）
```

## 🛠 核心碳核算技术实现与闭环逻辑

区别于普通的 UI 表单展示，本系统在 `shared-mock-data.js` 中构建了一套高保真算力追踪引擎，在不依赖真实后端的情况下，跨模块打通了**“数据源 -> 活动数据(AD) -> 核算模型 -> 排放因子(EF) -> GWP 版本 -> 碳排放总量(tCO₂e) 闭环”**：

### 1. 碳核算核心公式链条
系统完全重现了标准的物料流与因子法核算：
$$\text{Emission } (\text{tCO}_2\text{e}) = \frac{\text{活动数据 (AD)} \times \text{排放因子 (EF)} \times \text{全球变暖潜势 (GWP)}}{1000}$$
*   **固定燃烧（厂区锅炉/柴油发电机）**：点对点打通天然气量（$Nm^3$）、柴油量（$L$），根据工业指南自动换算低位热值及折标煤核算。
*   **能源间接（外购电力）**：与生态环境部最新电网因子库对齐，实现市场基准法与位置基准法级联核算。

### 2. 多页间 Iframe 跨窗口数据流同步机制
系统内部实现了一套基于 **HTML5 跨域/同源事件机制** 触发的响应式事件总线：
*   **实时总线同步**：当用户在 `datasource.html` 手工补录或更改点位状态，或在 `gwp.html` 中一键切换系统当前的活性标准（如将基准从 IPCC 第五次报告 AR5 切换到最新第六次报告 AR6），系统会自动派发 `carbon-mock-updated` 全局同源同步事件。
*   **全链响应**：汇总壳体 `index.html` 以及各活跃子模块监听到 `storage` 事件后，会立即就地重新运行前端 `calculateEmission` 计算链路。数据质量（条目完整率、准确率、及时率）及看板卡片上的总量、核算成熟度等级（L1 $\rightarrow$ L3）会自动发生数值上的**滚动式动态更新**。

### 3. 高质量企业级版本控制与审计流（Audit Trail）
*   **模型与 GWP 的多版本对比**：在 `model.html` 和 `gwp.html` 中集成了全量异常数据源清单和版本对比面板，直观通过高亮（Highlight）展示两个标准之间的因子变化率与变化趋势。
*   **配置回滚控制**：支持历史配置版本的一键回滚，并能保障回滚记录可作为新副本再次写入审计流，完美复刻真实碳盘查（Carbon Audit）中的追溯链合规评分规范。
